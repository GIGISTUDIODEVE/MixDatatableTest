Item ↔ Object 아키텍처 관리

Item_Object_Architecture.md에서 정의한 “아이템 = 권리/재료, 오브젝트 = 월드 실체” 원칙을 그대로 Firestore에 반영하기 위한 CRUD 도구입니다. Def(정적 설계)와 User-owned Entity(동적 상태)를 분리하여 저장/불러오기/삭제를 수행합니다.

Item = 권리/재료/트리거 · Object = 위치/타이머/레벨을 가진 실체
정적 Def: buildingDefs · plantDefs · islandDefs · itemDefs · upgradeDefs
동적 상태: users/{uid}/inventory · islands · worldObjects

아이템에는 위치/타이머/성장 단계가 절대 포함되지 않도록 JSON을 분리하고, Entity에는 레벨/타이머/생산 상태만 유지합니다.

정적 Def 관리

건물/식물/섬/아이템/업그레이드 Def는 상태를 가지지 않는 설계 데이터입니다. collection을 선택하고 JSON을 수정 후 저장하세요.

※ 비용/시간/조건/효과 등 게임 밸런스 파라미터만 Def에 유지하고, 상태 값은 저장하지 않습니다.

collection docId 요약

유저 상태 관리 · Item을 통한 권리

users/{uid}/inventory는 “권리/재료/트리거”만 저장합니다. 위치/타이머/레벨을 절대 넣지 말고, itemDefId와 quantity만 유지합니다.

docId itemDefId quantity

유저 상태 관리 · Island Entity

섬은 항상 Entity로 관리합니다. 해금 구역, 개발도, 소속 오브젝트 리스트 등 장기 상태만 저장합니다.

docId islandDefId level unlockedZones

유저 상태 관리 · World Object Entity

건물/식물은 worldObjects에 Entity로 저장합니다. 위치/타이머/레벨/생산 상태만 포함하고, 구매권은 inventory 아이템으로 분리합니다.

docId type defId level/stage position
