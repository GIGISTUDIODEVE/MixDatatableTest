Firebase 서버 몬스터 타입 관리

MonsterDataTable.md에 정의된 서버 몬스터 타입(`monsterTypes`)을 Firestore에서 조회하고 신규 등록합니다.

1) 서버 몬스터 타입 조회

상위 컬렉션 `monsterTypes`에서 이름 순으로 정렬해 가져옵니다. 서버 공용 데이터만 조회합니다.

불러온 데이터 미리보기(JSON)

`baseStats`, `growthPerLevel`를 포함한 전체 문서 내용을 JSON으로 확인하여 정상 저장 여부를 검증합니다.

monsterTypeId name rarityTier tags dataVersion createdAt

2) 몬스터 타입 생성

MonsterDataTable.md 필수 필드를 만족하도록 서버 전역 타입을 생성합니다. createdAt/updatedAt은 서버 타임스탬프로 기록됩니다.

스탯 프리셋 불러오기

미리 정의한 baseStats·growthPerLevel 프리셋을 선택해 입력값을 빠르게 채웁니다. 필요 시 아래 "3) 스탯 프리셋 관리"에서 프리셋을 추가하세요.

MonsterDataTable.md에 정의된 스탯 키만 입력 가능합니다. 모든 값은 숫자여야 합니다. 스탯 카테고리(생존/공격/방어·유틸/쿨·자원)별 고정 입력란으로 제공됩니다.

레벨당 성장치도 동일한 스탯 키에 대해 숫자로 입력합니다. 카테고리별 고정 필드에 동일 키를 입력해야 합니다.

MonsterDataTable.md 기준 스탯 키만 허용되며 숫자값을 입력해야 합니다.
필수 필드: monsterTypeId, name, tags, rarityTier, baseStats, growthPerLevel, createdAt

3) 스탯 프리셋 관리

기본 baseStats·growthPerLevel을 미리 정의해 Firestore `monsterStatPresets` 컬렉션에 저장하고, 몬스터 생성 시 불러와 사용할 수 있습니다.

MonsterDataTable.md의 모든 스탯 키가 필수이며 숫자만 입력 가능합니다.

baseStats (필수, 고정 스탯 필드)

growthPerLevel (필수, 고정 스탯 필드)

저장된 프리셋

서버에 저장된 프리셋 목록을 확인하고, 몬스터 생성 시 상단 선택 박스에서 불러올 수 있습니다.

presetId name createdAt updatedAt
